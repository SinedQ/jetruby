Класс Router служит для создания маршрутов и управления ими. Когда вы добавляете маршрут, объект Router сохраняет информацию о пути, 
HTTP-методе и обработчике для данного маршрута. При обращении к конкретному маршруту, Router определяет соответствующий обработчик и выполняет его.

Причина использования ключевого слова extend для модуля Resource связана с тем, что класс Router не наследуется от класса Resource, а использует его методы и функциональность. 
С использованием extend, модуль Resource расширяет функциональность самого класса Router, позволяя ему использовать методы из модуля напрямую, без создания экземпляра класса Resource.
Таким образом, класс Router может получить доступ к методам модуля, как если бы они были его собственными методами.
